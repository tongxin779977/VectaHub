# VectaHub 1.0 产品规划路线图

> 版本: 1.0.0 | 最后更新: 2026-05-03 | 状态: 执行中

---

## 前言

本文档基于用户测试报告和实际使用需求，定义了 VectaHub 1.0 版本的高中优先级功能规划。

**当前问题**：
- 用户测试通过率仅 38%（20/53）
- 大量命令选项缺失
- 工作流开发调试效率低

---

## 🎯 第一阶段：核心功能补全（高优先级）

### 目标：用户测试通过率 ≥ 90%

---

#### 1.1 命令选项一致性修复

**问题**：测试显示很多预期的选项和子命令不存在

| 问题 | 影响 | 优先级 |
|------|------|--------|
| `run` 命令缺少 `--dry-run` 选项 | NL 测试全部失败 | P0 |
| `doctor` 命令缺少 `--verbose` 选项 | 诊断信息不完整 | P1 |
| `security` 缺少 `status`、`policy` 子命令 | 功能不完整 | P1 |
| `audit` 缺少 `list` 子命令 | 与用户预期不符 | P1 |
| `mode` 缺少 `show` 子命令（或改进交互） | 可用性问题 | P1 |
| `history` 命令参数验证过严 | 用户体验差 | P1 |
| `dev module` 缺少 `list` 子命令 | 功能不完整 | P1 |

**验收标准**：
- [ ] `vectahub run --dry-run` 工作正常
- [ ] `vectahub doctor --verbose` 显示详细诊断
- [ ] 所有 53 个测试用例通过率 ≥ 90%
- [ ] 所有命令帮助信息完整准确

**预计工作量**：1-2 天

---

#### 1.2 工作流调试功能

**问题**：复杂工作流开发效率低，缺少调试能力

**功能需求**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 单步执行 | 按步骤逐个执行，可暂停 | P0 |
| 断点设置 | 在特定步骤设置断点 | P0 |
| 中间变量查看 | 执行过程中查看上下文变量 | P0 |
| 执行日志增强 | 详细的步骤执行日志 | P1 |
| 调试模式开关 | `--debug` 选项启用调试 | P1 |

**使用示例**：

```bash
# 调试模式执行工作流
vectahub run -f workflow.yaml --debug

# 单步执行
vectahub run -f workflow.yaml --step-by-step
```

**验收标准**：
- [ ] 支持单步执行模式
- [ ] 支持断点设置（在 YAML 中标记）
- [ ] 调试过程中可查看上下文变量
- [ ] 调试日志完整清晰

**预计工作量**：2-3 天

---

#### 1.3 工作流版本控制

**问题**：工作流修改没有历史记录，无法回滚

**功能需求**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 自动版本保存 | 每次保存工作流自动创建版本 | P0 |
| 版本列表 | 查看工作流的所有历史版本 | P0 |
| 版本回滚 | 回滚到任意历史版本 | P0 |
| 版本对比 | 对比两个版本的差异 | P1 |
| 版本备注 | 为版本添加说明 | P1 |

**CLI 设计**：

```bash
# 查看工作流版本历史
vectahub history <workflow-id> --versions

# 回滚到特定版本
vectahub workflow rollback <workflow-id> --version <version-id>

# 对比版本差异
vectahub workflow diff <workflow-id> --from <v1> --to <v2>
```

**验收标准**：
- [ ] 每次保存自动创建新版本
- [ ] 可查看完整版本历史
- [ ] 可回滚到任意版本
- [ ] 可对比版本差异

**预计工作量**：2-3 天

---

#### 1.4 预设工作流模板库

**问题**：用户需要从零开始创建，缺少常用场景模板

**功能需求**：

| 模板类型 | 包含内容 | 优先级 |
|----------|----------|--------|
| Git 工作流 | commit → push → 部署 | P0 |
| CI/CD | install → test → build → deploy | P0 |
| 数据备份 | 压缩 → 加密 → 上传 | P0 |
| 文件处理 | 批量重命名 → 转换格式 | P1 |
| 部署流程 | 构建 → 测试 → 发布 | P1 |
| 系统监控 | 检查资源 → 告警 | P1 |

**CLI 设计**：

```bash
# 查看可用模板
vectahub templates list

# 使用模板创建工作流
vectahub templates use git-workflow --name my-workflow

# 保存当前工作流为模板
vectahub templates save my-workflow --name custom-template
```

**验收标准**：
- [ ] 内置 5+ 常用模板
- [ ] 支持查看和使用模板
- [ ] 支持将现有工作流保存为模板
- [ ] 模板参数化配置

**预计工作量**：2 天

## ⚡ 第二阶段：体验提升（中优先级）

#### 2.1 工作流导入导出

**功能需求**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 导出为 YAML | 导出工作流为 YAML 文件 | P0 |
| 导出为 JSON | 导出工作流为 JSON 格式 | P1 |
| 导入工作流 | 从文件导入工作流 | P0 |
| 批量导出 | 一次导出多个工作流 | P1 |

**CLI 设计**：

```bash
# 导出工作流
vectahub export <workflow-id> --output my-workflow.yaml

# 导入工作流
vectahub import my-workflow.yaml --name imported-workflow

# 批量导出
vectahub export --all --output-dir ./workflows/
```

**验收标准**：
- [ ] 支持 YAML/JSON 格式导出
- [ ] 支持从文件导入
- [ ] 导入的工作流可正常执行
- [ ] 支持批量操作

**预计工作量**：1-2 天

#### 2.2 错误恢复机制

**功能需求**：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 断点续传 | 从失败的步骤继续执行 | P0 |
| 重试策略 | 配置步骤失败重试次数 | P0 |
| 失败通知 | 工作流失败时发送通知 | P1 |
| 错误汇总 | 生成错误报告 | P1 |

**YAML 配置示例**：

```yaml
workflow:
  name: example
  retry:
    max_attempts: 3
    delay_seconds: 5
  on_failure: continue  # continue | abort

steps:
  - id: step1
    type: exec
    retry:
      max_attempts: 5
    on_failure: abort
```

**CLI 设计**：

```bash
# 从断点继续执行
vectahub resume <execution-id>

# 查看失败的执行
vectahub history --failed
```

**验收标准**：
- [ ] 支持从失败步骤继续
- [ ] 支持步骤级重试配置
- [ ] 支持工作流级重试策略
- [ ] 失败后可查看详细错误信息

**预计工作量**：2-3 天
