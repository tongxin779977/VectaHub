## 1. 项目简介

本项目是基于现代前端工具链构建的 React 单页应用 (SPA)。
- **核心框架**: [React 19](https://react.dev/) (v19.2.4) —— 提供了更简洁的 API 和性能改进。
- **构建工具**: [Vite 8](https://vite.dev/) (v8.0.1) —— 下一代前端构建工具，提供极速的热重载 (HMR) 和构建体验。
- **开发语言**: JavaScript (JSX)
- **代码规范**: [ESLint 9](https://eslint.org/) —— 采用扁平配置文件 (`eslint.config.js`)。

## 3. 开发入门

### 3.1 依赖安装
在项目根目录下运行以下指令以安装所有必要依赖：
```bash
npm install
```

### 3.2 本地开发
启动极速本地开发服务器：
```bash
npm run dev
```
默认访问地址: `http://localhost:5173`

### 3.3 项目构建
将应用打包到 `dist` 目录，准备进行生产环境部署：
```bash
npm run build
```

### 3.4 代码检查
运行 ESLint 扫描并检测潜在的代码问题：
```bash
npm run lint
```

## 4. 技术栈详情

### React 19
- **新特性**: 支持 `use` Hook 处理异步数据，增强了对 Actions 的支持，简化了表单处理逻辑。
- **渲染引擎**: 默认使用 React 19 的新并发渲染特性。

### Vite 8
- **极速预构建**: 使用 esbuild 快速处理依赖。
- **环境变量**: 通过 `.env` 文件配置，在代码中通过 `import.meta.env` 访问。
- **插件系统**: 已预装 `@vitejs/plugin-react`，支持 HMR 及 Fast Refresh。

## 5. 代码规范与最佳实践

- **命名规范**: 
  - 组件文件采用 **PascalCase** (如 `MyComponent.jsx`)。
  - 普通 JS 文件及工具函数采用 **camelCase** (如 `utils.js`)。
  - 样式文件建议与组件同名 (如 `App.jsx` 对应 `App.css`)。
- **状态管理**: 
  - 简单场景推荐使用 React 原生的 `Context API`。
  - 复杂场景推荐集成 `Zustand` 或 `Redux Toolkit`。
- **路由**: 
  - 建议集成 `react-router-dom` (v7+) 进行页面导航管理。

## 6. 后续扩展建议

1. **引入 TypeScript**: 随着项目规模扩大，建议逐步迁移至 `.tsx` 以获得类型安全。
